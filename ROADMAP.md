# ROADMAP: PulseEngine — Тренажер Намерения

**Концепция:** Клиентское веб-приложение. Движок строит непрерывный график на основе криптографически стойкого хаоса (энтропии). Единственный способ взаимодействия — намерение и сознание пользователя. Никаких кликов, курсоров или внешних данных. Проект проприетарный, автономный, с глубокой геймификацией вероятностей.

---

### Фаза Архитектуры и Математической Честности
*Слой, гарантирующий, что хаос не искажен плохим кодом, лагами процессора или утечками памяти.*

- **Переписать генератор энтропии: запрашивать батч из `crypto.getRandomValues` за один вызов, чтобы исключить узкие места процессора и гарантировать 100% честность рандома.**
- **Внедрить `Math.fround` для всех физических вычислений, чтобы физика работала идентично (детерминированно) на любых процессорах от старых Android до Mac M3.**
- **Отслеживать `document.hidden` (Page Visibility API) для жесткой паузы физики при свернутой вкладке — предотвращает накопление мусорного времени (`dt`) и искажение статистики.**
- Защитить `CONFIG` и `POLICY` через `Object.freeze()` после старта для предотвращения рантайм-мутаций.
- Реализовать защиту от "Спирали смерти": аппаратно ограничить максимальный `dt`, чтобы график не улетал в бесконечность при системных лагах ОС.
- Устранить скачок `dt` при первой отрисовке, используя точный старт от `performance.now()`.
- Добавить метод `destroy()` для сборки мусора и полной очистки памяти при сбросе сессии.
- Добавить строгий режим `"use strict";` и изолировать глобальную область `window` через IIFE.

### Фаза Гипнотической Плавности (Рендеринг)
*Слой визуального совершенства. Любой лаг разрушает концентрацию, поэтому график должен рисоваться с идеальной частотой обновления.*

- **Заменить жесткие отрезки `lineTo` на квадратичные кривые `quadraticCurveTo`. Линия должна быть идеально гладкой на любой скорости, чтобы вводить в состояние потока.**
- **Перенести весь рендеринг в Web Worker через `OffscreenCanvas`. UI браузера больше никогда не прервет отрисовку.**
- **Написать алгоритм бинарного поиска для Culling'а: рендерить только те точки, которые сейчас физически находятся на экране (гарантия 120 FPS).**
- **Предварительно вычислять палитру цветов (Color LUT) при старте, полностью удалив тяжелую интерполяцию RGB из цикла отрисовки.**
- Заменить `window.resize` на `ResizeObserver`, повешенный прямо на `<canvas>`.
- Кешировать вызовы `ctx.strokeStyle` и `ctx.lineWidth` — обновлять графический контекст только при реальном изменении дельты.
- Динамически пересчитывать `bufferSize` на лету, исходя из ширины экрана и скорости, а не хардкодить константу.
- Реализовать градиентную заливку (`createLinearGradient`) хвоста вместо плоских отрезков.

### Фаза Автономности (PWA и Стабильность)
*Слой, превращающий веб-страницу в надежное приложение, независимое от интернета.*

- **Интегрировать Wake Lock API: экран не должен гаснуть ни при каких обстоятельствах, пока открыт тренажер.**
- **Блокировать все системные жесты браузера (свайпы назад, pull-to-refresh) через CSS `touch-action: none` и `overscroll-behavior`.**
- **Настроить Service Worker для кэширования движка. Тренажер должен мгновенно запускаться в авиарежиме.**
- Сохранять массив истории графика в `IndexedDB` в реальном времени (защита от случайного закрытия вкладки).
- Добавить `manifest.json` для установки на домашний экран телефона/ПК без рамок браузера.
- Поддержка Fullscreen API по двойному тапу для полного погружения.
- Сохранять пользовательские пресеты цветов в локальном хранилище.

### Фаза Геймификации Намерения (Интеграция Октализа)
*Слой смыслов. Превращаем наблюдение за рандомом в дофаминовый цикл, не добавляя ни одной кнопки.*

- **Внедрить фоновый расчет Z-score (стандартного отклонения). Движок должен математически фиксировать моменты, когда линия держится в центре вопреки теории вероятностей — это засчитывается как "Аномалия Намерения" (Эпическое Значение).**
- **Создать визуальную обратную связь "Потока": при длительной аномалии линия начинает светиться (Bloom), цвета теплеют. Пользователь понимает, что у него получается, только через визуал (Развитие и Обратная связь).**
- **Жесткий срыв Потока: если хаос берет верх и линия бьется о край экрана, свечение моментально гаснет, счетчик сбрасывается. Инстинктивное желание избежать потери визуала (Потери и Избегание).**
- **Ограничить "Глубокую Синхронизацию" 15 минутами в день. Защита от выгорания и создание дефицита — каждую сессию начнут ценить (Дефицит и Нетерпение).**
- Автоматически сохранять "Слепки" (Seeds) в галерею пользователя, если энтропия выдает вероятность 0.001% — например, идеальную синусоиду (Собственность и Владение).
- Ввести систему "Рангов", которая растет только за счет накопленных секунд математических аномалий, а не просто за время работы программы (Развитие и Достижения).
- Зашить крайне редкие визуальные пасхалки (вспышки золотого цвета), шанс появления которых стремится к нулю (Непредсказуемость и Любопытство).

### Будущие Горизонты (Визионерство)
*Глобальные фичи, которые ждут своей очереди после шлифовки базы.*

- **Синхронный мультиплеер и Глобальный Рейтинг.** Наблюдение за тем, как сотни людей в реальном времени прогибают вероятности, и выявление самых сильных "Исказителей".
- **Генеративный Эмбиент (Web Audio API).** Движок синтезирует звук, напрямую привязанный к кривизне и напряжению линии.
- **Интеграция с реальным миром (IoT).** Передача аномалий на умный свет в комнате пользователя (Philips Hue / DMX).
- Искусственный интеллект, пытающийся предсказать и "подыграть" (или наоборот, усложнить) паттерны пользователя.
